## 背景

在使用Python异步函数（`async`）的过程中，`async` 函数具有非常高的传染性，导致所有调用 `async` 函数的调用者也必须是 `async` 函数。

问题出现在：某些`__init__`函数中也需要调用`async`函数，但是python并不提供异步的`__init__`函数。

此时可以使用python的`__await__`魔术方法实现。如下：

## 使用 `__await__` 实现异步初始化

根据Python官方文档，当`await`一个**对象**时，实际会执行对象的 `__await__` 方法，借此，我们实现了一个类似于异步的初始化函数。

```python
class AsyncMixin:
    def __init__(self, *args, **kwargs):
        """
        Do not override.
        """
        self.__storedargs = args, kwargs
        self.async_initialized = False

    async def __ainit__(self, *args, **kwargs):
        pass

    async def __initobj(self):
        assert not self.async_initialized
        self.async_initialized = True
        await self.__ainit__(*self.__storedargs[0], **self.__storedargs[1])
        return self

    def __await__(self):
        return self.__initobj().__await__()
```

```python
class AsyncExample(AsyncMixin):
    async def __ainit__(self, param1, param2):
        await self.async_operation1(param1)
        await self.async_operation2(param2)

    async def async_operation1(self, param):
        await asyncio.sleep(1)
        print(f"Async operation 1 with param: {param}")

    async def async_operation2(self, param):
        await asyncio.sleep(2)
        print(f"Async operation 2 with param: {param}")


async def main():
    await AsyncExample("Parameter 1", "Parameter 2")


if __name__ == "__main__":
    import asyncio

    asyncio.run(main())
```

## 更好的做法

其实根据python官方文档，python的官方`async`支持出现的比较晚，此时市面上已经有一些第三方的异步框架存在（深入了解Python异步编程的读者可能知道，在`async`出现之前，其实Python可以使用`yield`实现异步）。

python的 `__await__` 方法，是为第三方异步框架的兼容性而设计的。并不建议非框架开发者在开发中实际使用这个方法。所有在这里，只是记录存在这种方法，但是并不推荐使用。

实际使用过程中

1. 可以通过**依赖注入**的方法，将`__init__`函数中需要异步初始化的内容，在一个`async`函数中初始化好，传入`__init__`中去，这样就避免了`__init__`中执行异步函数。
2. 或者实现自定义的async初始化工厂方法，将需要异步初始化的对象初始化好，传入`__init__`中去。

二者的代码完全类似，只是编程思想的区别。
